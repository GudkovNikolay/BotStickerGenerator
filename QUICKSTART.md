# Быстрый старт

## Шаг 1: Установка зависимостей

```bash
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

## Шаг 2: Настройка конфигурации

1. Создайте файл `.env` в корне проекта
2. Заполните минимально необходимые переменные:

```env
BOT_TOKEN=ваш_токен_от_BotFather

# Локальный Stable Diffusion (рекомендуется для MVP)
IMAGE_BACKEND=local
LOCAL_SD_URL=http://127.0.0.1:7860

# (Опционально) Hugging Face Inference API
# IMAGE_BACKEND=hf
# HF_TOKEN=ваш_huggingface_token
# HF_MODEL=stabilityai/stable-diffusion-xl-base-1.0
```

Минимально необходимые переменные для локального режима:
- `BOT_TOKEN` - получите у @BotFather в Telegram
- `LOCAL_SD_URL` - URL локального Stable Diffusion (по умолчанию `http://127.0.0.1:7860`)

## Шаг 3: Запуск бота

```bash
python bot.py
```

Бот запустится и начнет отвечать на команды в Telegram.

## Первые команды

1. Откройте вашего бота в Telegram
2. Отправьте `/start`
3. Отправьте `/generate` и следуйте инструкциям
4. Введите описание стикеров (например: "Кот в космосе")
5. Дождитесь генерации и получите стикеры!

## Структура проекта

- `bot.py` - главный файл запуска
- `handlers.py` - обработчики команд и сообщений
- `database.py` - модели базы данных
- `db_service.py` - сервис для работы с БД
- `image_generator.py` - генерация изображений через Hugging Face Inference API (или локальный SD)
- `sticker_processor.py` - обработка изображений в стикеры
- `payment_service.py` - сервис платежей (заглушка для MVP)
- `config.py` - конфигурация приложения
- `utils.py` - вспомогательные функции

## Важные замечания

- Для работы требуется активный интернет-соединение
- Hugging Face Inference API может иметь ограничения/free-tier (зависит от аккаунта и модели)
- База данных SQLite создается автоматически при первом запуске
- Логи сохраняются в файл `bot.log`

## Работа с базой данных

### Просмотр содержимого базы данных

База данных SQLite хранится в файле `bot.db` в корне проекта. Для просмотра данных можно использовать:

#### Вариант 1: SQLite CLI (встроен в систему)

```bash
# Открыть интерактивную консоль SQLite
sqlite3 bot.db

# Полезные команды в SQLite:
.tables                    # Показать все таблицы
.schema users              # Показать структуру таблицы users
SELECT * FROM users;       # Показать всех пользователей
SELECT * FROM generations; # Показать все генерации
SELECT * FROM payments;    # Показать все платежи
.quit                     # Выйти
```

#### Вариант 2: Python скрипт (рекомендуется)

В проекте уже есть готовый скрипт `check_db.py`:

```bash
python check_db.py
```

Он покажет:
- Всех пользователей с их данными
- Последние 20 генераций
- Статистику по статусам генераций
- Последние 10 платежей
- Последние 10 реферальных наград

Это самый удобный способ быстро посмотреть содержимое БД.

#### Вариант 3: Визуальные инструменты

- **DB Browser for SQLite** (https://sqlitebrowser.org/) - графический интерфейс
- **VS Code расширение**: SQLite Viewer
- **DBeaver** - универсальный инструмент для БД

### Очистка базы данных

⚠️ **Внимание**: Это удалит все данные!

```bash
# Остановите бота (Ctrl+C)
# Удалите файл базы данных
rm bot.db

# При следующем запуске бот создаст новую пустую БД
python bot.py
```

### Резервное копирование

```bash
# Создать копию базы данных
cp bot.db bot.db.backup

# Восстановить из резервной копии
cp bot.db.backup bot.db
```

## Перезапуск приложения

### Обычный перезапуск

1. Остановите бота нажав `Ctrl+C` в терминале
2. Запустите снова:
   ```bash
   python bot.py
   ```

### Перезапуск после исправления ошибок

Если вы исправили ошибки в коде или изменили структуру базы данных:

1. **Остановите бота** (`Ctrl+C`)

2. **Если изменилась структура БД**, может потребоваться пересоздать базу:
   ```bash
   # Удалите старую БД (⚠️ потеряете данные!)
   rm bot.db
   ```

3. **Запустите бота заново**:
   ```bash
   python bot.py
   ```

4. **Проверьте логи** на наличие ошибок:
   ```bash
   tail -f bot.log  # Linux/Mac
   # или просто откройте bot.log в редакторе
   ```

### Перезапуск с очисткой временных файлов

Если возникли проблемы с генерацией изображений:

```bash
# Остановите бота
# Удалите временные файлы
rm -rf temp_images/* generated_stickers/*
# Запустите снова
python bot.py
```

## Решение проблем

**Ошибка "Ошибка загрузки конфигурации":**
- Убедитесь, что файл `.env` существует и содержит все необходимые переменные

**Ошибка при генерации изображений:**
- Проверьте, что `HF_TOKEN` правильный и имеет доступ к Inference API
- Попробуйте другой `HF_MODEL` (некоторые модели могут быть недоступны/заняты)

**Ошибка "index already exists" или проблемы с БД:**
- Удалите базу данных: `rm bot.db`
- Запустите бота снова - он создаст новую БД с правильной структурой

**Бот не отвечает:**
- Проверьте, что `BOT_TOKEN` правильный
- Убедитесь, что бот запущен (`python bot.py`)
- Проверьте логи в файле `bot.log`
