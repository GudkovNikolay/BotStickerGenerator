# Telegram Bot для генерации стикеров

MVP Telegram-бота на Python для генерации наборов стикеров по текстовому промпту.

## Возможности

- 🎨 Генерация стикеров по текстовому описанию через **Hugging Face Inference API** (Stable Diffusion/SDXL)
- 🎁 Система бесплатных генераций для новых пользователей
- 👥 Реферальная система с бонусами
- 💳 Базовая структура для интеграции платежей (ЮKassa/Stripe)
- 📊 Статистика пользователей и логирование генераций
- 🗄️ База данных SQLite (легко мигрировать на PostgreSQL)

## Технологии

- **Python 3.10+**
- **Aiogram 3.x** - современный асинхронный фреймворк для Telegram ботов
- **SQLAlchemy** - ORM для работы с базой данных
- **Локальный Stable Diffusion (Automatic1111 / др.)** - основная генерация изображений
- **Hugging Face Inference API** (опционально) - облачная генерация (например, SDXL)
- **Pillow (PIL)** - обработка изображений в стикеры WebP
- **SQLite** - база данных (для MVP)

## Установка

1. Клонируйте репозиторий:
```bash
git clone <repository_url>
cd BotStickerGenerator
```

2. Создайте виртуальное окружение:
```bash
python -m venv venv
source venv/bin/activate  # для Windows: venv\Scripts\activate
```

3. Установите зависимости:
```bash
pip install -r requirements.txt
```

4. Создайте файл `.env` в корне проекта и заполните минимальные переменные:

```env
BOT_TOKEN=your_bot_token_here

# Локальный Stable Diffusion (рекомендуется для MVP)
IMAGE_BACKEND=local
LOCAL_SD_URL=http://127.0.0.1:7860

# (Опционально) Hugging Face Inference API
# IMAGE_BACKEND=hf
# HF_TOKEN=your_huggingface_token_here
# HF_MODEL=stabilityai/stable-diffusion-xl-base-1.0
```

Для локального режима минимум:
- `BOT_TOKEN` - токен вашего Telegram бота (получите у `@BotFather`)
- `LOCAL_SD_URL` - URL локального SD WebUI / API (по умолчанию `http://127.0.0.1:7860`)

## Команды бота

- `/start` - Начать работу с ботом
- `/grid` - Создать сетку стикеров по текстовому описанию
- `/stats` - Показать статистику пользователя
- `/referral` - Реферальная система
- `/buy` - Купить генерации (в разработке)
- `/help` - Справка по командам

## Использование

1. Запустите бота командой `/start`
2. Используйте `/grid` для создания сетки стикеров
3. Отправьте текстовое описание стикеров (например: "Кот в космосе")
4. Дождитесь генерации и обработки изображений
5. Получите готовые стикеры в формате WebP

## Реферальная система

Каждый пользователь получает уникальный реферальный код. При регистрации нового пользователя по реферальной ссылке:
- Реферер получает +1 бесплатную генерацию
- Новый пользователь получает стандартный набор бесплатных генераций

## Платежи

В MVP режиме платежи не реализованы полностью. Структура готова для интеграции:
- **ЮKassa** - для пользователей из РФ
- **Stripe** - для международных платежей

## База данных

Используется SQLite для простоты MVP. Модели:
- `users` - пользователи бота
- `generations` - история генераций
- `payments` - платежи
- `referral_rewards` - реферальные награды

Для миграции на PostgreSQL измените `DATABASE_URL` в `.env`:
```
DATABASE_URL=postgresql+asyncpg://user:password@localhost/dbname
```

## Логирование

Все события логируются в файл `bot.log` и в консоль.
